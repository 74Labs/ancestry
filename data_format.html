Data format

Basic format

The data for lineage plot should be arranged as an array of node objects. Each object should have following properties:

  • name (string) Name of the node.
  • parent (string | null) Name of the parent node.
  • x (number) Value x (applicable only in lineage-scatter plot).
  • y (number) Value y (applicable only in lineage-scatter plot).
  • z (optional: number) Value for the heatmap layer.
  • date (optional: number - UNIX timestamp) If specified, plot will have time axis and the property will be used as x value. Applicable only in lineage-plot.
  • inLinkLabel (string | null) Name of the ingoing link.
  • type (optional: string) Type of the node.
  • series (optional: string) Name of the series the node belongs to.
  • selected (optional: boolean) Specifies whether a node should be selected on render.
  • tooltip (optional: array of strings) Array with strings corresponding to lines in a tooltip. If not used, name property is used for tooltip.

Phylogenetic tree format

This plot requires slightly more complex data format. It takes an array of tree node objects. A tree node should consist of:

  • length (number) Length of the branch with respect to the parent tree node.
  • taxon (null | object) If null, then the node is non-leaf and children property must be defined. Otherwise, the node is a leaf and the object should have the same format as a lineage plot's node except parent property which is not needed here.
  • children (null | two element array) If taxon is defined, then children must be either not defined or set to null. If taxon is null, children should be an array containing two other tree nodes.

The required data format can be generated using Neighbor Joining NPM package based on a distance matrix.
