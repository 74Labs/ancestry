Ancestry

Ancestry is an AngularJS module consisting of a number of directives allowing to generate interactive lineage plots based on d3.js.

To install this package with NPM use: npm install ancestry

Plots

There are currently 4 types of plots offered by Ancestry:
  • Lineage
  • Lineage-scatter
  • Radial lineage
  • Radial phylogenetic tree
All plots support series, legend and heatmap layer. The plots are generated by passing a configuration object as an attribute to a directive. The object should consist of two properties:
  • data
  • layout
The role of these attributes is self-explanatory, but their form might differ between particular plots and will be described in sections below.

Lineage

Lineage plot uses a horizontal tree layout. It can keep multiple trees (meaning that trees have distinct roots).

Data format

The data for lineage plot should be arranged as an array of node objects. Each object should have following properties:
  • name (string) Name of the node.
  • parent (string | null)Name of the parent node.
  • z (optional: number) Value for the heatmap layer.
  • inLinkLabel (string | null)Name of the ingoing link.
  • type (optional: string)Type of the node.
  • series (optional: string) Name of the series the node belongs to.
  • selected (optional: boolean) Specifies whether a node should be selected on render.
  • tooltip (optional: array of strings) Array with strings corresponding to lines in a tooltip. If not used, name property is used for tooltip.

Plot-specific layout

Lineage plot is equipped with an axis which can be configured as described below.

  • axis (object)
    • title (string || default: "")The title of the (horizontal) axis.
    • show (true | false || default: true)Determines whether the axis is drawn.
    • gridOnly (true | false || default: false)Determines whether only the grid is displayed (axis and ticks are hidden). Note: Only works when axis.show is set to true.
  • minGenerationWidth (number || default: 50)Specifies the minimum width of a generation (distance between a parent and its children nodes) in pixels.

Radial-lineage

Radial-lineage plot is very similar to the lineage plot. Its main difference is that it uses a radial layout. It can also hold more nodes avoiding overlap, but on the other side does not allow for zooming and panning. Another advantage is that it can show all the nodes on the path between a node and its root aligned when a node is clicked. Currently, radial plot does not allow for node selection.

Radial phylogenetic tree

This plot allows to show one or more rooted phylogenetic trees in a radial layout.

  • nodeClick (array) Callback with injected arguments ($event - click event Object, $node - clicked node's data)

Data format

This plot requires slightly more complex data format. It takes an array of tree node objects. A tree node should consist of:
  • length (number) Length of the branch with respect to the parent tree node.
  • taxon (null | object)If null, then the node is non-leaf and children property must be defined. Otherwise, the node is a leaf and the object should have the same format as a lineage plot's node except parent property which is not needed here.
  • children (null | two element array) If taxon is defined, then children must be either not defined or set to null. If taxon is null, children should be an array containing two other tree nodes.
The required data format can be generated using Neighbor Joining NPM package based on a distance matrix.

Plot-specific layout

In radial-lineage plot, size of the leaf nodes' labels can be controlled.

  • outerNodeLabel (object || default: {"font-size": 14, "font-family": "Roboto,Helvetica Neue,sans-serif"})Object with SVG text attributes defining appearance of leaf nodes' labels.
  • showLeafNodes (false | true || default: false) Determines whether leaf nodes are shown.

General layout

  • title (string || default: "")The title of the plot.
  • width (number | null || default: null)Width of the plot container. If null, width is set to width of DOM element the directive is in.
  • height (number | null || default: 600)Height of the plot container. If null, height is set to height of DOM element the directive is in.
  • margin Extra space excluding special features such as title, legend, etc.
    • left (number || default: 10)Size of the left margin.
    • right (number || default: 10)Size of the right margin.
    • top (number || default: 10)Size of the top margin.
    • bottom (number || default: 10)Size of the bottom margin.
  • plotPadding In lineage and lineage-scatter plots: allows to specify lineage padding to prevent the lineage from occupying the whole plotting area. If x or y is null, the padding in x or y direction will be adjusted to fit node labels. If x or y is a number, the padding will equal to the given number (in pixels). In radial-lineage and phylogenetic-tree, x and y are not discriminated and maximum of these is taken as plot padding leading to shrinking the lineages equally in both directions.
    • x (number | null || default: null)Width of plot padding in direction x.
    • y (number | null || default: null)Height of plot padding in direction y.
  • backgroundColour (string || default: "none")Background colour of the plot. If "none", background is transparent.
  • textColour (string || default: "black")Colour of all text elements in the plot.
  • nodeTypes (object || default: {}) Dictionary with node types. Node types should be string keys, values should be objects with attributes of SVG circle such as "stroke-width", "r" etc. Changing colours should be avoided as it is also controlled by series and could interfere.
  • seriesColours (object || default: null) Object with series ids as keys and colours as values. If this property is used, it is requried to specify series id for each node data object.
  • nodeLabel (object || default: {"font-size": 12, "dy": 4, "font-family": "Roboto,Helvetica Neue,sans-serif"})Object with SVG text attributes defining appearance of node labels
  • linkLabel (object || default: {"font-size": 12, "dy": 4, "font-family": "Roboto,Helvetica Neue,sans-serif"})Object with SVG text attributes defining appearance of link labels
  • link (object || default: {"stroke": "#ccc", "stroke-width": 1})Object with SVG path attributes defining appearance of links between nodes
  • linkCaptureWidth (number || default: 10)Determines the width of invisible link placed on top of the visible link to facilitate convenient mouse events.
  • showLinkArrowhead (false | true)Specifies whether links end with arrowheads. By default, this is true only in lineage-scatter plot.
  • labelCollisionDetection (object) Only available for lineage and lineage-scatter plots. Label collision detection takes care of hiding colliding labels so that the plot stays clean and readable.
    • enabled ("never" | "onInit" | "onEveryChange" | "onDelay" || default: "never") If "never", the collision detection will be switched off and all labels will be shown at all times. If "onInit", the collisions will be handled only once - during plot initialization. If "onEveryChange", the collisions will be resolved upon every zoom or pan operation. In case there are numerous nodes, it might introduce too much lag as many computations have to be done upon each change. If "onDelay", the collision detection will trigger only after an interval specified as updateDelay from the last zoom/pan action. This might be the best option for visualizations with a couple of hundreds of nodes.
    • updateDelay (number || default: 500) Determines the number of milliseconds from last zoom/pan update when collision detection should trigger. Only relevant when enabled is set to "onDelay".
    • order (object || default: {"nodeLabel": 1, "linkLabel": 1})Object specifying priority in performing collision detection. All elements with priority 1 are resolved at the same time and before elements with priority 2.
  • groupSelection (object) Group selection allows for selecting multiple nodes at once.
    • enabled (false | true || default: false) Determines whether group selection is enabled.
    • selectionRectangle (object || default: {"stroke-width": 1, "stroke-dasharray": 4, "rx": 3, "ry": 3, "stroke": "steelblue" }) Determines the attributes of SVG rect which is used for group selection.
  • heatmap (object) Heatmap adds an extra information layer for nodes with z property defined.
    • enabled (false | true || default: false) Determines whether heatmap is enabled.
    • title (string | null || default: null) Title of the heatmap displayed with the colour bar.
    • colorScale (array of two element arrays || default: [[0, "#008ae5"], [1, "yellow"]]) Determines the colour range used for the heatmap. Currently, the array should have form [[0, colour1], [1, colour2]], where colour1 and colour2 correspond respectively to minimum and maximum values in heatmap layer.
    • colorBar (object)
      • show (false | true || default: true) Determines whether heatmap's colour bar is shown.
      • height (number (of pixels) | string with percentage || default: "90%") Determines the height of the colour bar. If a number is given, the colour bar will be that many pixels high, if a string with percentage is given, its height will be relative to the total plot height.
      • width (number (of pixels) | string with percentage || default: 30) Determines the height of the colour bar. If a number is given, the colour bar will be that many pixels wide, if a string with percentage is given, its width will be relative to the total plot width.
      • padding (object)
        • left (number || default: 10) Determines the padding between the plot area and the color bar).
        • right (number || default: 0) Determines the padding between the color bar and the legend.
    • circle (object || default: {"r": 16}) Determines the attributes of SVG circle which is used as heatmap circle layer behind the nodes.
    • opacity (number || default: 0.4) Determines the opacity of heatmap circles and the colour bar.
  • legend (object) Legend can be used to show and hide particular group of nodes that have the same series property.
    • show (false | true || default: false) Determines whether legend is shown.
    • x (number || default: 1) Determines the horizontal position of the legend with reference to the plot area. 0 refers to the left border of the plot area and 1 refers to the right border of the plot area.
    • y (number || default: 0.5) Determines the vertical position of the legend with reference to the plot area. 0 refers to the top border of the plot area and 1 refers to the bottom border of the plot area.
    • anchor (object)
      • x ("left" | "center" | "right"|| default: "left") Determines the horizontal anchor point of the legend frame.
      • y ("top" | "center" | "bottom" || default: "center") Determines the vertical anchor point of the legend frame.
    • orientation ("horizontal" | "vertical" || default: "vertical") Determines the direction in which the legend is laid out.
    • backgroundColour (null | string || default: null) If null, legend inherits plot's background colour. If string, specific colour is used.
  • tooltip (object)
    • show (true | false || default: true) Determines whether tooltip for nodes is shown.
    • showSeriesBar (true | false || default: false) Determines whether a small horizontal bar indicating the series the mouseovered node belongs to is displayed.
    • align ("left" | "center" | "right" || default: "left") Determines the text align inside tooltip.
  • brush (object)
    • margin
      • top (number || default: 20)Size of the margin above the bush panel.
      • bottom (number || default: 0)Size of the margin below the brush panel.
    • height (number || default: 200)Height of the bush panel.
    • lockY (true | false || default: false)Disables brushing in Y direction. Useful especially when brush serves as a scroll bar.
    • boxRectangle (object || default: {'stroke-width': 1, stroke: '#aaa'})Specifies the layout of the border of the brush panel which is an SVG rect element.
    • drawTrees (true | false || default: true)Determines whether trees (nodes and links) are rendered in the brush panel area.
    • brushRectangleOnFullView (true | false || default: true)If true, when the full plot extent is shown, the brush selection rectangle is visible. If false, it disappears when the full plot extent is shown.
    • axis (object)Specifies the axis layout for the brush panel. Refer to the plot specific layout layout.axis for the specific configuration options.
  • nodeLabelPositions (array) The array specifies SVG text element positions relative to the corresponding node which are tried out in the given order in label collision detection. The array's elements should be objects with x, y, text-anchor properties.
    Default: [{x: 10, y: 0, "text-anchor": "left"}, {x: -10, y: 0, "text-anchor": "right"}]
  • controls (object) Determines the state of the control panel. show attribute specifies whether a control is included in the control panel. active determines the initial state of a control (on or off). position is used to determine the order of controls in the control panel.
    • download (object || default: {show: true, format: 'png', position: 0}) Control responsible for saving the plot. format can be 'png' or 'svg'.
    • zoom (object || default: {show: true, active: false, position: 1}) Control responsible for zooming and panning the plot.
    • brush (object || default: {show: true, active: false, position: 2}) Control responsible for showing brush plot below the original plot.
    • select (object || default: {show: true, active: false, position: 3}) Control responsible for enabling node selection mode.
    • label (object || default: {show: true, active: true, position: 4}) Control responsible for showing node and link labels.

Directive attributes

Input:
  • plot-data (object) Input for plot's data.
  • plot-layout (array) Input for plots configuration
Output:
  • nodes-selection Allows to attach action upon node selection event (both individual and group). Check out this example to see an example usage. Injected parameters:
    • $nodes (array of objects) Array of the node objects that are currently in the selection.
  • custom-node Allows to create custom nodes instead of default circles. Check out this example to see an example usage. Injected parameters:
    • $event ('draw' | 'update' | 'select') Specifies type of the event: 'draw' - nodes are being rendered for the first time. Upon this event, nodes' DOM elements should be appended to the $selection. 'update' - the nodes are redrawn upon zoom/pan/rotate actions (also called after 'draw'). It should be used to update nodes' positions. 'select' - node(s) has been selected. It should be used to mark the selected nodes in a specific way. This event is triggered only in select mode.
    • $selection (d3 selection) If $event is 'draw', $selection is an enter selection ready for creating new DOM elements. If 'update' or 'select', $selection is a normal selection holding all drawable nodes.
  • custom-link Allows to create custom links. Function should return a string corresponding to SVG path definition. Check out this example to see an example usage. Injected parameters:
    • $linkObject (object) Object representing a link.
  • node-click / node-mouse-over / node-mouse-out Allows to attach action upon node click / mouse over / mouse out events. Check out this example to see an example usage. Injected parameters:
    • $event (object) current d3 event object
    • $object (object) Corresponding node's object
  • link-click / link-mouse-over / link-mouse-out Allows to attach action upon link click / mouse over / mouse out events. Check out this example to see an example usage. Injected parameters:
    • $event (object) current d3 event object
    • $object (object) Corresponding link's object

Navigable plot objects

All lineage plot elements (node / link / node label / link label) have object with contextual information. For example, node object stores not only the input data that was used to create it, but it also keeps a reference to its DOM object, parent and children objects, ingoing link object and the node's x and y position. Node and link objects are injected as parameters in node/link click/mouse over/mouse out events. This makes it easy to create interactions with the plot. For example, upon a node click, the whole path leading through all ancestor nodes can be highlighted by traversing all the ancestor nodes iteratively. Most of the parameters available in the objects are listed below.

Node:
  • data (object) The original node object as specified by user in plot-data data input.
  • DOMElement (DOM) Node's DOM element.
  • parent (object) Parent node's object.
  • children (array) Array of children node objects.
  • inLink (object) Ingoing link's object.
  • label (object) Node's label object.
  • bbox (object) Bounding rectangle for label collision detection purposes.
  • depth (number) Generation index of the node.
  • x (number) Current position x of the node.
  • y (number) Current position y of the node.
  • _x (number) Original position x of the node (after creating d3 tree layout).
  • _y (number) Current position y of the node (after creating d3 tree layout).
Link:
  • DOMElement (DOM) Links's DOM element (SVG's path).
  • sourceNode (object) Object of the node that the link originated from.
  • targetNode (object) Object of the node that the link goes to.
  • label (object) Link's label object.
Node's label:
  • DOMElement (DOM) Node label's DOM element (SVG's text).
  • text (string) Text of the label.
  • node (object) Corresponding node's object.
  • currentLabelPos (string) Determines the current position of the label with reference to the node. One of objects from layout.nodeLabelPositions array.
  • x (number) Current position x of the label (anchored according to currentLabelPos).
  • y (number) Current position y of the label.
  • dy (number) Vertical offset of the label.
  • width (number) Width of the label.
  • height (number) Height of the label.
  • bbox (object) Bounding rectangle for label collision detection purposes.
  • isColliding (boolean) Specifies whether collisions has been detected for the label.
Link's label:
  • DOMElement (DOM) Link label's DOM element (SVG's text).
  • text (string) Text of the label.
  • link (object) Corresponding link's object.
  • x (number) Current position x of the label (anchored at the center).
  • y (number) Current position y of the label.
  • dy (number) Vertical offset of the label.
  • width (number) Width of the label.
  • height (number) Height of the label.
  • bbox (object) Bounding rectangle for label collision detection purposes.
  • isColliding (boolean) Specifies whether collisions has been detected for the label.
