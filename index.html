Ancestry

Ancestry is an AngularJS module consisting of a number of directives allowing to generate interactive lineage plots based on d3.js.

To install this package with NPM use: npm install ancestry

Plots

There are currently 4 types of plots offered by Ancestry:
  • Lineage
  • Lineage-scatter
  • Radial lineage
  • Radial phylogenetic tree
All plots support series, legend and heatmap layer. The plots are generated by passing a configuration object as an attribute to a directive. The object should consist of two properties:
  • data
  • layout
The role of these attributes is self-explanatory, but their form might differ between particular plots and will be described in sections below.

Lineage

Lineage plot uses a horizontal tree layout. It can keep multiple trees (meaning that trees have distinct roots).

Attributes

  • value (object) Input for plot configuration JSON.
  • selectedNodes (array) Callback with injected arguments ($nodes - array of names of the selected nodes)
  • nodeClick (array) Callback with injected arguments ($event - click event Object, $node - clicked node's data)

Data format

The data for lineage plot should be arranged as an array of node objects. Each object should have following properties:
  • name (string) Name of the node.
  • parent (string | null)Name of the parent node.
  • z (optional: number) Value for the heatmap layer.
  • type (optional: string)Type of the node.
  • series (optional: string) Name of the series the node belongs to.
  • tooltip (optional: array of strings) Array with strings corresponding to lines in a tooltip. If not used, name property is used for tooltip.

Plot-specific layout

Lineage plot is equipped with an axis which can be configured as described below.

axis (object)
  • title (string || default: "")The title of the (horizontal) axis.
  • show (true | false || default: true)Determines whether the axis is drawn.
  • gridOnly (true | false || default: false)Determines whether only the grid is displayed (axis and ticks are hidden). Note: Only works when axis.show is set to true.

Lineage-scatter

Data format

The data for lineage-scatter plot should be arranged as an array of node objects. Each object should have following properties:
  • name (string) Name of the node.
  • parent (string | null)Name of the parent node.
  • x (number)Value x
  • y (number)Value y
  • z (optional: number) Value for the heatmap layer.
  • type (optional: string)Type of the node.
  • series (optional: string) Name of the series the node belongs to.
  • tooltip (optional: array of strings) Array with strings corresponding to lines in a tooltip. If not used, name property is used for tooltip.

Plot-specific layout

Lineage plot is equipped with an axis which can be configured as described below.

  • xAxis (object)
    • title (string || default: "")Title of the horizontal axis.
    • format (string | null || default: null)Determines the format of tick values of the horizontal axis. String should match d3's format specification. Value null results in general d3 format.
  • yAxis (object)
    • title (string || default: "")Title of the vertical axis.
    • format (string | null || default: null)Determines the format of tick values of the vertical axis. String should match d3's format specification. Value null results in general d3 format.

Radial-lineage

Radial-lineage plot is very similar to the lineage plot. Its main difference is that it uses a radial layout. It can also hold more nodes avoiding overlap, but on the other side does not allow for zooming and panning. Another advantage is that it can show all the nodes on the path between a node and its root aligned when a node is clicked. Currently, radial plot does not allow for node selection.

    • Radial phylogenetic tree

    This plot allows to show one or more rooted phylogenetic trees in a radial layout.

    • Plot
    • JSON

    Directive

    Attributes

    • value (object) Input for plot configuration JSON.
    • branchlength (boolean) Determines whether real branch lengths are shown.
    • nodeClick (array) Callback with injected arguments ($event - click event Object, $node - clicked node's data)

    Data format

    This plot requires slightly more complex data format. It takes an array of tree node objects. A tree node should consist of:
    • length (number) Length of the branch with respect to the parent tree node.
    • taxon (null | object)If null, then the node is non-leaf and children property must be defined. Otherwise, the node is a leaf and the object should have the same format as a lineage plot's node except parent property which is not needed here.
    • children (null | two element array) If taxon is defined, then children must be either not defined or set to null. If taxon is null, children should be an array containing two other tree nodes.
    The required data format can be generated using Neighbor Joining NPM package based on a distance matrix.

    Plot-specific layout

    In radial-lineage plot, size of the leaf nodes' labels can be controlled.

    • outerNodeLabel (object || default: {"font-size": 14, "font-family": "Roboto,Helvetica Neue,sans-serif"})Object with SVG text attributes defining appearance of leaf nodes' labels.
    • showLeafNodes (false | true || default: false) Determines whether leaf nodes are shown.

    General layout

    • title (string || default: "")The title of the plot.
    • width (number | null || default: null)Width of the plot container. If null, width is set to width of DOM element the directive is in.
    • height (number | null || default: 600)Height of the plot container. If null, height is set to height of DOM element the directive is in.
    • margin Extra space excluding special features such as title, legend, etc.
      • left (number || default: 10)Size of the left margin.
      • right (number || default: 10)Size of the right margin.
      • top (number || default: 10)Size of the top margin.
      • bottom (number || default: 10)Size of the bottom margin.
    • backgroundColour (string || default: "none")Background colour of the plot. If "none", background is transparent.
    • textColour (string || default: "black")Colour of all text elements in the plot.
    • nodeTypes (object || default: {}) Dictionary with node types. Node types should be string keys, values should be objects with attributes of SVG circle such as "stroke-width", "r" etc. Changing colours should be avoided as it is also controlled by series and could interfere.
    • nodeLabel (object || default: {"font-size": 12, "font-family": "Roboto,Helvetica Neue,sans-serif"})Object with SVG text attributes defining appearance of node labels
    • link (object || default: {"stroke": "#ccc", "stroke-width": 1})Object with SVG path attributes defining appearance of links between nodes
    • maxZoom (number not smaller than 1 || default: 10)Determines maximum zoom. If no zooming is intended, this should be set to 1.
    • labelCollisionDetection (object) Only available for lineage and lineage-scatter plots. Label collision detection takes care of hiding colliding labels so that the plot stays clean and readable.
      • enabled ("never" | "onInit" | "onEveryChange" | "onDelay" || default: "never") If "never", the collision detection will be switched off and all labels will be shown at all times. If "onInit", the collisions will be handled only once - during plot initialization. If "onEveryChange", the collisions will be resolved upon every zoom or pan operation. In case there are numerous nodes, it might introduce too much lag as many computations have to be done upon each change. If "onDelay", the collision detection will trigger only after an interval specified as updateDelay from the last zoom/pan action. This might be the best option for visualizations with a couple of hundreds of nodes.
      • updateDelay (number || default: 500) Determines the number of milliseconds from last zoom/pan update when collision detection should trigger. Only relevant when enabled is set to "onDelay".
    • groupSelection (object) Only available for lineage and lineage-scatter plots.Group selection allows for selecting multiple nodes at once. To start the selection, hold CTRL key and press left mouse button and perform a drag choosing desired area. All the selected nodes are externally available through selectedNodes attribute.
      • enabled (false | true || default: false) Determines whether group selection is enabled.
      • selectionRectangle (object || default: {"stroke-width": 1, "stroke-dasharray": 4, "rx": 3, "ry": 3, "stroke": "steelblue" }) Determines the attributes of SVG rect which is used for group selection.
    • heatmap (object) Heatmap adds an extra information layer for nodes with z property defined.
      • enabled (false | true || default: false) Determines whether heatmap is enabled.
      • title (string | null || default: null) Title of the heatmap displayed with the colour bar.
      • colourScale (array of two element arrays || default: [[0, "#008ae5"], [1, "yellow"]]) Determines the colour range used for the heatmap. Currently, the array should have form [[0, colour1], [1, colour2]], where colour1 and colour2 correspond respectively to minimum and maximum values in heatmap layer.
      • colourBar (object)
        • show (false | true || default: true) Determines whether heatmap's colour bar is shown.
        • height (number (of pixels) | string with percentage || default: "90%") Determines the height of the colour bar. If a number is given, the colour bar will be that many pixels high, if a string with percentage is given, its height will be relative to the total plot height.
        • width (number (of pixels) | string with percentage || default: 30) Determines the height of the colour bar. If a number is given, the colour bar will be that many pixels wide, if a string with percentage is given, its width will be relative to the total plot width.
        • position ("right" || default: "right") Determines the position of the colour bar in reference to the chart. Currently, the only supported position is "right".
      • circle (object || default: {"r": 16}) Determines the attributes of SVG circle which is used as heatmap circle layer behind the nodes.
      • opacity (number || default: 0.4) Determines the opacity of heatmap circles and the colour bar.
    • legend (object) Legend can be used to show and hide particular group of nodes that have the same series property.
      • show (false | true || default: false) Determines whether legend is shown.
      • position (object)
        • x ("left" | "center" | "right" || default: "right") Determines the horizontal position of the legend with reference to the chart.
        • y ("top" | "center" | "bottom" || default: "center") Determines the vertical position of the legend with reference to the chart.
      • anchor (object)
        • x ("inside" | "outside" || default: "outside") Determines the horizontal anchor of the legend with reference to the chart.
        • y ("inside" | "outside" || default: "outside") Determines the vertical anchor of the legend with reference to the chart.
      • orientation ("horizontal" | "vertical" || default: "vertical") Determines the direction in which the legend is laid out.
      • backgroundColour (null | string || default: null) If null, legend inherits plot's background colour. If string, specific colour is used.
    • tooltip (object) Only available for lineage and lineage-scatter plots. Tooltip provides basic information about a node on hover. Tooltip contents are customizable through nodes' tooltip property.
      • show (false | true || default: true) Determines whether tooltip is displayed.
      • showSeriesBar (false | true || default: false) Determines whether a small colour bar corresponding to series colour will be added to tooltips.
      • align ("left" | "center" | "right" || default: "left") Determines how the text in tooltip is aligned.
