General layout

  • title (string || default: "") The title of the plot.
  • width (number | null || default: null) Width of the plot container. If null, width is set to width of DOM element the directive is in.
  • height (number | null || default: 600) Height of the plot container. If null, height is set to height of DOM element the directive is in.
  • margin Extra space excluding special features such as title, legend, etc.
    • left (number || default: 10) Size of the left margin.
    • right (number || default: 10) Size of the right margin.
    • top (number || default: 10) Size of the top margin.
    • bottom (number || default: 10) Size of the bottom margin.
  • plotPadding In lineage and lineage-scatter plots: allows to specify lineage padding to prevent the lineage from occupying the whole plotting area. If x or y is null, the padding in x or y direction will be adjusted to fit node labels. If x or y is a number, the padding will equal to the given number (in pixels). In radial-lineage and phylogenetic-tree, x and y are not discriminated and maximum of these is taken as plot padding leading to shrinking the lineages equally in both directions.
    • x (number | null || default: null) Width of plot padding in direction x.
    • y (number | null || default: null) Height of plot padding in direction y.
  • backgroundColor (string || default: "none") Background color of the plot. If "none", background is transparent.
  • textColor (string || default: "black") Color of all text elements in the plot.
  • nodeTypes (object || default: {}) Dictionary with node types. Node types should be string keys, values should be objects with attributes of SVG circle such as "stroke-width", "r" etc. Changing colors should be avoided as it is also controlled by series and could interfere.
  • seriesColors (object || default: null) Object with series ids as keys and colors as values. If this property is used, it is requried to specify series id for each node data object.
  • nodeLabel (object || default: {"font-size": 12, "dy": 4, "font-family": "Roboto,Helvetica Neue,sans-serif"} ) Object with SVG text attributes defining appearance of node labels
  • linkLabel (object || default: {"font-size": 12, "dy": 4, "font-family": "Roboto,Helvetica Neue,sans-serif"} ) Object with SVG text attributes defining appearance of link labels
  • link (object || default: {"stroke": "#ccc", "stroke-width": 1}) Object with SVG path attributes defining appearance of links between nodes
  • linkCaptureWidth (number || default: 10) Determines the width of invisible link placed on top of the visible link to facilitate convenient mouse events.
  • showLinkArrowhead (false | true) Specifies whether links end with arrowheads. By default, this is true only in lineage-scatter plot.
  • labelCollisionDetection (object) Label collision detection takes care of hiding colliding labels so that the plot stays clean and readable.
    • enabled ("never" | "onInit" | "onEveryChange" | "onDelay" || default: "never") If "never", the collision detection will be switched off and all labels will be shown at all times. If "onInit", the collisions will be handled only once - during plot initialization. If "onEveryChange", the collisions will be resolved upon every zoom or pan operation. In case there are numerous nodes, it might introduce too much lag as many computations have to be done upon each change. If "onDelay", the collision detection will trigger only after an interval specified as updateDelay from the last zoom/pan action. This might be the best option for visualizations with a couple of hundreds of nodes.
    • updateDelay (number || default: 500) Determines the number of milliseconds from last zoom/pan update when collision detection should trigger. Only relevant when enabled is set to "onDelay".
    • order (object || default: {"nodeLabel": 1, "linkLabel": 1}) Object specifying priority in performing collision detection. All elements with priority 1 are resolved at the same time and before elements with priority 2.
  • groupSelection (object) Group selection allows for selecting multiple nodes at once.
    • enabled (false | true || default: false) Determines whether group selection is enabled.
    • selectionRectangle (object || default: {"stroke-width": 1, "stroke-dasharray": 4, "rx": 3, "ry": 3, "stroke": "steelblue"} ) Determines the attributes of SVG rect which is used for group selection.
  • heatmap (object) Heatmap adds an extra information layer for nodes with z property defined.
    • enabled (false | true || default: false) Determines whether heatmap is enabled.
    • title (string | null || default: null) Title of the heatmap displayed with the color bar.
    • colorScale (array of two element arrays || default: [[0, "#008ae5"], [1, "yellow"]]) Determines the color range used for the heatmap. Currently, the array should have form [[0, color1], [1, color2]], where color1 and color2 correspond respectively to minimum and maximum values in heatmap layer.
    • colorBar (object)
      • show (false | true || default: true) Determines whether heatmap's color bar is shown.
      • height (number (of pixels) | string with percentage || default: "90%") Determines the height of the color bar. If a number is given, the color bar will be that many pixels high, if a string with percentage is given, its height will be relative to the total plot height.
      • width (number (of pixels) | string with percentage || default: 30) Determines the height of the color bar. If a number is given, the color bar will be that many pixels wide, if a string with percentage is given, its width will be relative to the total plot width.
      • padding (object)
        • left (number || default: 10) Determines the padding between the plot area and the color bar).
        • right (number || default: 0) Determines the padding between the color bar and the legend.
      • circle (object || default: {"r": 16}) Determines the attributes of SVG circle which is used as heatmap circle layer behind the nodes.
      • opacity (number || default: 0.4) Determines the opacity of heatmap circles and the color bar.
  • legend (object) Legend can be used to show and hide particular group of nodes that have the same series property.
    • show (false | true || default: false) Determines whether legend is shown.
    • x (number || default: 1) Determines the horizontal position of the legend with reference to the plot area. 0 refers to the left border of the plot area and 1 refers to the right border of the plot area.
    • y (number || default: 0.5) Determines the vertical position of the legend with reference to the plot area. 0 refers to the top border of the plot area and 1 refers to the bottom border of the plot area.
    • anchor (object)
      • x ("left" | "center" | "right"|| default: "left") Determines the horizontal anchor point of the legend frame.
      • y ("top" | "center" | "bottom" || default: "center") Determines the vertical anchor point of the legend frame.
    • orientation ("horizontal" | "vertical" || default: "vertical") Determines the direction in which the legend is laid out.
    • backgroundColor (null | string || default: null) If null, legend inherits plot's background color. If string, specific color is used.
  • tooltip (object) Tooltip will only work if directive attributes node-mouse-over and node-mouse-out are not supplied.
    • show (true | false || default: true) Determines whether tooltip for nodes is shown.
    • showSeriesBar (true | false || default: false) Determines whether a small horizontal bar indicating the series the mouseovered node belongs to is displayed.
    • align ("left" | "center" | "right" || default: "left") Determines the text align inside tooltip.
  • brush (object)
    • margin
      • top (number || default: 20) Size of the margin above the bush panel.
      • bottom (number || default: 0) Size of the margin below the brush panel.
    • height (number || default: 200) Height of the bush panel.
    • lockY (true | false || default: false) Disables brushing in Y direction. Useful especially when brush serves as a scroll bar.
    • boxRectangle (object || default: {'stroke-width': 1, stroke: '#aaa'}) Specifies the layout of the border of the brush panel which is an SVG rect element.
    • drawTrees (true | false || default: true) Determines whether trees (nodes and links) are rendered in the brush panel area.
    • brushRectangleOnFullView (true | false || default: true) If true, when the full plot extent is shown, the brush selection rectangle is visible. If false, it disappears when the full plot extent is shown.
  • nodeLabelPositions (array) The array specifies SVG text element positions relative to the corresponding node which are tried out in the given order in label collision detection. The array's elements should be objects with x, y, text-anchor properties.
    Default: [{x: 10, y: 0, "text-anchor": "left"}, {x: -10, y: 0, "text-anchor": "right"}]
  • controls (object) Determines the state of the control panel. show attribute specifies whether a control is included in the control panel. active determines the initial state of a control (on or off). position is used to determine the order of controls in the control panel.
    • download (object || default: {show: true, format: 'png', position: 0}) Control responsible for saving the plot. format can be 'png' or 'svg'.
    • zoom (object || default: {show: true, active: false, position: 1}) Control responsible for zooming and panning the plot.
    • brush (object || default: {show: true, active: false, position: 2}) Control responsible for showing brush plot below the original plot.
    • select (object || default: {show: true, active: false, position: 3}) Control responsible for enabling node selection mode.
    • label (object || default: {show: true, active: true, position: 4}) Control responsible for showing node and link labels.
